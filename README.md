# sh

Miniature shell for Linux written in C with `PATH` lookup, `cd` support, pipelines, and environment variable expansion.

## Task

Вам предлагается написать небольшой аналог `shell`, который частично реализован в [файле](src/template.c)

### Требования к реализации

1. **Парсинг и раскрытие путей:**
   - Реализовать функцию `parse`, разбивающую промпт по пробельным символам на отдельные токены (`dst`) с ограничением `MAX_TOKENS`.
   - Поддержать автоматическое раскрытие символа домашней директории (`~` или `~/...`), заменяя его на путь из переменной окружения `HOME`.

2. **Раскрытие переменных окружения:**
   - Реализовать функцию `expand_single_token` для посимвольного разбора токенов и подстановки значений переменных окружения вида `$VAR`.

3. **Запуск и выполнение процессов (`run`):**
   - Использовать `fork` и `execvp` для запуска внешних программ.
   - Поддержать пайплайны, перенаправление файловых дескрипторов ввода-вывода можно реализовать с помощью системных вызовов `pipe` и `dup2`.

4. **Встроенные команды:**
   - Реализовать встроенные команды `exit`, `cd` и `export`.
   - При вызове встроенной команды `export` без аргументов выводить текущие переменные окружения (`extern char **environ`). При вызове с аргументом вида `NAME=VALUE` — записывать их через вызов `setenv`.

5. **Отображение текущей директории (cwd):**
   - Динамически отслеживать текущую рабочую директорию с помощью `getcwd` и форматировать в виде: `sh (in /path/to/cwd)>`.

6. **Отсутствие утечек памяти:**
   - Все динамические токены, созданные на этапе парсинга и раскрытия переменных, должны гарантированно освобождаться на каждой итерации цикла шелла с помощью функции `free_tokens`.

```bash
make run
```
